⚡6 หลักการพื้นฐาน (ไม่เปลี่ยนตาม LV)

เป็นสันกระดูกของสกิล — บทเรียนทั้งหมดขยายจาก 6 ข้อนี้
หลักที่ 1
ภาพที่ match กันพอใช้ดีกว่าภาพสวยที่ไม่ match — คุมให้ทุก shot ดูเป็นโลกเดียวกันก่อนค่อยทำให้สวย
หลักที่ 2
ความกำกวม = โมเดลเลือกเอง ไม่ใช่คุณ — ทุกรายละเอียดที่ไม่ได้บอก = สุ่ม จงบอกทุกอย่างที่สำคัญ
หลักที่ 3
Reference ภาพไม่ใช่ของเสริม — เป็นพื้นฐาน — โปรเจกต์หลายช็อตต้องออกแบบ ref chain ก่อนเจนภาพแรก
หลักที่ 4
ตัวเลข > คำคุณศัพท์ — "2.1 เมตร" ไม่ใช่ "ยาว" · "85mm" ไม่ใช่ "telephoto" · "40 คอลัมน์" ไม่ใช่ "เยอะ"
หลักที่ 5
Negative-state ความล้มเหลวที่พบบ่อย — "no age markers, no text, no watermark, no [failure mode ที่เคยเจอ]"
หลักที่ 6
Self-review ไม่ใช่ optional — เป็นขั้นสุดท้ายของการเจน ไม่ใช่ "nice-to-have"

L09
ถ้าต้อง lock element เฉพาะ — crop ภาพ ref ให้เห็นแค่ element นั้น
Validated 2.5/3 meso

ตัวอย่าง: ถ้าจะให้วิทยุตัวนี้โผล่ในทุกช็อต — ครอปวิทยุจากภาพแรกไว้ แล้วส่ง "ภาพครอป" เป็น ref · ห้ามส่งภาพฉากร้านทั้งภาพ
L12
ถ้าช็อตกว้างกว่าครอป — ใช้ dual-ref (ครอป + ภาพฉากเต็ม)
เพิ่งสร้าง Pending 0/3 meso
L09 ครอปช่วย lock element แต่ background รอบๆ สูญเสีย ไปด้วย (โมเดล "คิดใหม่") · แก้โดยส่ง 2 refs: ครอป + ภาพฉากเต็ม
ค้นพบได้ไง: หลัง L09 ผ่าน arena · คุณรีวิวภาพ 3A แล้วเห็นว่าฉากเปลี่ยนไป (ไม่มีพีระมิดแล้ว!) · S04 ทดสอบ 6 generations พิสูจน์ว่า dual-ref ดีกว่าจริง
L05
Ref มุมหลังตัวละคร ใช้ lock หน้าไม่ได้
Pending 0/3 meso
ตัวละครที่โผล่หลายมุม (หน้า/ข้าง/หลัง/ในตู้) ต้อง มี anchor ภาพต่อมุม · ภาพ back-view บอกแค่เสื้อผ้า/ท่า ไม่บอกหน้า
ตัวอย่าง: DHYANA มีภาพแม่มุมหลัง (หุม) ใช้เป็น ref · ตอนจะเจนหน้าแม่ในโพด → หน้าออกมาคนละคน (model ไม่เห็นหน้าจาก back-view)
L11
Tier-1 refs สำหรับวิดีโอ ต้องออกแบบให้เข้ากันระหว่าง ref ด้วย
Pending 0/5 macro
ภาพ ref สวยแยกกันไม่พอ — ใน video มันต้อง interpolate ระหว่างกัน · pose/angle/bg ต้อง compatible ทุกคู่ที่จะใช้รวมกัน
ตัวอย่าง: DHYANA ผีเสื้อ + มือพระ · ภาพมือในท่า mudra vs ภาพยื่นมือ → Seedance interpolate ไม่ได้ → anatomy break

L03
Opaque vs transparent — ต้องใช้คำศัพท์ตรงกันทุกช็อต
Pending 1/3 meso
คำว่า "frosted panels" กับ "opaque body with face-window" → โมเดลเจนต่างกันคนละแบบ · ต้อง ล็อคคำศัพท์ design-language ใช้เหมือนกันทุกที่
ตัวอย่าง: DHYANA pod — บางช็อตใช้ "transparent panels" บางช็อตใช้ "frosted" → บางช็อตตัวโปร่ง บางช็อตตัวทึบ → ไม่ match
L08
Palette ใน world-time เดียวกัน ต้อง state ชัดเจน
Pending 1/3 meso
ฉาก "flashback อดีต" กับ "ปัจจุบัน" ต้องตั้งใจให้ palette ต่าง — แต่ ภายใน flashback ทุกช็อต ต้องตรงกัน · "dusk" ไม่พอ · ต้องใช้ "warm pink fading to amber"
ตัวอย่าง: DHYANA #04 ฉาก memory (warm amber) vs #08 ฉากเดียวกัน (cool blue) → ดูเหมือนคนละสถานที่

🏟️Arena System — ระบบทดสอบที่ทำให้เลเวลอัปได้เอง

ระบบใหม่ที่เราสร้างใน LV 2.2 — จำลองโปรเจกต์สังเคราะห์เพื่อ validate lesson โดยไม่ต้องมีโปรเจกต์จริง

Lesson จะ promote จาก pending → validated ได้ต่อเมื่อ: meso tier = 2 passes ข้าม ≥2 projects · macro tier = 3 passes ข้าม ≥3 projects · Arena สร้าง "projects สังเคราะห์" ที่ต่างจาก DHYANA เดิม ≥3 แกน (จาก 9 แกน: character/era/genre/palette/model/modality/culture/scale/domain)

🎲
Generator
ออกแบบ scenario ใหม่ให้ต่างจากโปรเจกต์เดิม ≥3 แกน + เขียน pass criteria ล็อคไว้ก่อน
🖌️
Runner (blinded)
ทำงานเหมือน client จริง ไม่รู้ว่าทดสอบ lesson ไหน · เรียก API สร้างภาพจริง
⚖️
Judge
ให้คะแนนผลลัพธ์ตาม criteria ที่ล็อคไว้ · ห้ามแก้ criteria ย้อนหลัง
🕵️
Skeptic (anti-Goodhart)
ตั้งคำถาม "test ยากพอไหม" "lesson จำเป็นจริงไหม" · void trial ได้แม้ Judge ให้ pass
จุดแข็งของระบบ: 4 roles ต้อง "เห็นด้วย" ถึงจะนับเป็น pass · Skeptic คือกำแพงกัน Goodhart (เราไม่อยาก validate lesson ปลอมเพราะ scenario ออกแบบง่ายเกิน) · ทุก scenario ต้อง flip ≥3 แกน จาก baseline เดิมเพื่อให้เป็น "project ต่าง" จริง
4 trials ที่รันไปแล้ว:
  • S01 (ร้านของชำ 1920s) → void · Skeptic จับได้ว่า text บรรยายเยอะเกิน แยกไม่ออกว่าเพราะ crop หรือเพราะ text
  • S02 (tsuba ญี่ปุ่น) → partial · มี A/B control + text น้อยลง · 3A ชนะ 3B 2/4 dims
  • S03 (หัวนักบวช Aztec) → PASS · 4 dimensions เป็น directional/asymmetric · 3A ชนะ 3/4 · L09 promote
  • S04 (Victorian orrery) → findings · 6-gen multi-factor · H1/H2/H3 pass + H4 partial · L12 เกิด
